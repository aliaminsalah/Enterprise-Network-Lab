Campus Network & ISP Implementation

Complete CLI configuration including VTPv3, HSRP, ISP integration, and step-by-step verification commands to ensure full network reachability.

R1

Edge Router
Performs inter-VLAN routing, NAT (if needed later), and DHCP services. OSPF is integrated with the Core layer. 
! --- Basic Configuration ---
enable
configure terminal
hostname R1
no ip domain-lookup

! --- DHCP Server Configuration ---
ip dhcp excluded-address 10.0.0.1 10.0.0.10
ip dhcp excluded-address 10.0.0.128 10.0.0.138

ip dhcp pool VLAN10_CLIENTS
 network 10.0.0.0 255.255.255.128
 default-router 10.0.0.126
 exit

ip dhcp pool VLAN20_VOICE
 network 10.0.0.128 255.255.255.192
 default-router 10.0.0.190
 exit

! --- Interface Configuration ---
ipv6 unicast-routing

interface Loopback0
 ip address 10.0.128.1 255.255.255.255
 ipv6 address 2001:DB8:0:128::1/128
 ipv6 ospf 1 area 0
 exit

interface Ethernet0/0
 description LINK_TO_CSW1
 ip address 10.0.131.1 255.255.255.252
 ipv6 address 2001:DB8::1:1/127
 ipv6 ospf 1 area 0
 no shutdown
 exit

interface Ethernet0/1
 description LINK_TO_CSW2
 ip address 10.0.131.5 255.255.255.252
 ipv6 address 2001:DB8::2:1/127
 ipv6 ospf 1 area 0
 no shutdown
 exit

interface Ethernet0/2
 description LINK_TO_ISP
 ip address 172.16.1.1 255.255.255.252
 no shutdown
 exit

! --- Routing Configuration ---
ip route 0.0.0.0 0.0.0.0 172.16.1.2

router ospf 1
 router-id 10.0.128.1
 network 10.0.128.1 0.0.0.0 area 0
 network 10.0.131.0 0.0.0.3 area 0
 network 10.0.131.4 0.0.0.3 area 0
 default-information originate
 exit

🛠 R1 Troubleshooting & Verification

  • show ip ospf neighborVerify OSPF adjacency with CSW1 & CSW2. State should be FULL.
  • show ip route ospfVerify R1 is learning internal subnets (10.x.x.x) from the core.
  • ping 172.16.1.2Test Layer 3 physical connection to the ISP.
  • ping 8.8.8.8Test internet reachability (simulated Google DNS on ISP).

CSW1

Core / VTPv3 Server
Acts as the VTPv3 Server. Note: Run "vtp primary" in privileged EXEC mode to promote before creating VLANs. 
enable
configure terminal
hostname CSW1
no ip domain-lookup

! --- VTPv3 Server Configuration ---
vtp domain CAMPUS
vtp version 3
vtp mode server
vtp password cisco123
exit
! Run 'vtp primary' in EXEC mode here!
configure terminal

! --- VLAN Creation ---
vlan 10
 name CLIENTS
vlan 20
 name VOICE
vlan 30
 name PRINTERS
vlan 40
 name MANAGEMENT
exit

! --- L3 Configuration & OSPF ---
ip routing
ipv6 unicast-routing

interface Loopback0
 ip address 10.0.128.2 255.255.255.255
 exit

interface Ethernet0/0
 description LINK_TO_R1
 no switchport
 ip address 10.0.131.2 255.255.255.252
 no shutdown
 exit

router ospf 1
 router-id 10.0.128.2
 network 10.0.131.0 0.0.0.255 area 0
 exit

🛠 CSW1 Troubleshooting & Verification

  • show vtp statusVerify operating mode is Server and Primary ID is not all zeros.
  • show ip ospf neighborVerify connection to R1, CSW2, DSW1, DSW2.
  • show etherchannel summaryEnsure Port-channel 1 is up and ports are grouped (SU).
  • ping 8.8.8.8Test reachability to the external ISP network.

DSW1

Distribution / VTPv3 Client
Primary Root Bridge and Active HSRP Router for all VLANs. 
enable
configure terminal
hostname DSW1

! --- VTPv3 Client Configuration ---
vtp domain CAMPUS
vtp version 3
vtp mode client
vtp password cisco123

! --- Spanning Tree (Primary Root) ---
spanning-tree mode rapid-pvst
spanning-tree vlan 10,20,30,40 root primary

ip routing

! --- SVIs & HSRP Configuration ---
interface Vlan10
 ip address 10.0.0.1 255.255.255.128
 standby 10 ip 10.0.0.126
 standby 10 priority 110
 standby 10 preempt
 no shutdown
 exit

interface Vlan40
 ip address 10.0.0.209 255.255.255.248
 standby 40 ip 10.0.0.214
 standby 40 priority 110
 standby 40 preempt
 no shutdown
 exit

interface Vlan99
 ip address 10.0.131.11 255.255.255.248
 no shutdown
 exit

router ospf 1
 router-id 10.0.128.4
 network 10.0.0.0 0.0.0.255 area 0
 network 10.0.131.0 0.0.0.255 area 0
 exit

🛠 DSW1 Troubleshooting & Verification

  • show standby briefVerify DSW1 is 'Active' for HSRP groups 10 and 40.
  • show spanning-tree vlan 10Confirm this switch says "This bridge is the root".
  • show ip routeVerify OSPF default route (O*E2 0.0.0.0/0) is received from R1.
  • ping 10.0.131.1Ping the Gateway R1 to ensure internal routing is flawless.

ASW1

Access Layer
Access switch connecting End Devices. Default gateway points to the HSRP Virtual IP. 
enable
configure terminal
hostname ASW1

vtp domain CAMPUS
vtp version 3
vtp mode client
vtp password cisco123

interface Ethernet0/2
 description LINK_TO_PC
 switchport mode access
 switchport access vlan 10
 spanning-tree portfast edge
 spanning-tree bpduguard enable
 no shutdown
 exit

interface Vlan40
 ip address 10.0.0.211 255.255.255.248
 no shutdown
 exit

ip default-gateway 10.0.0.214

🛠 ASW1 Troubleshooting & Verification

  • show vlan briefVerify that VLANs 10, 20, 30, 40 have synced correctly via VTP.
  • show interfaces trunkEnsure the uplinks to DSW1/DSW2 are actively trunking.
  • ping 10.0.0.214Test management ping to the Distribution layer HSRP Gateway.

ISP Router

Internet Service Provider
Simulates the external internet. It contains Loopback interfaces acting as public websites and a static route pointing back to your enterprise network to ensure pings return successfully. 
enable
configure terminal
hostname ISP
no ip domain-lookup

! --- Link to R1 (Enterprise Edge) ---
interface Ethernet0/0
 description LINK_TO_CAMPUS_R1
 ip address 172.16.1.2 255.255.255.252
 no shutdown
 exit

! --- Simulated Public Internet Sites ---
interface Loopback0
 description GOOGLE_DNS
 ip address 8.8.8.8 255.255.255.255
 exit

interface Loopback1
 description CLOUDFLARE_DNS
 ip address 1.1.1.1 255.255.255.255
 exit
 
interface Loopback2
 description CISCO_UMBRELLA
 ip address 208.67.222.222 255.255.255.255
 exit

! --- Route Back to Campus Network ---
! (Allows ISP to reply to ping requests coming from internal VLANs)
ip route 10.0.0.0 255.0.0.0 172.16.1.1

end
write memory

🛠 ISP Troubleshooting & Verification

  • show ip routeVerify the static route to 10.0.0.0/8 exists in the routing table.
  • ping 172.16.1.1Test point-to-point physical connection to R1.
  • ping 10.0.128.1Test routing to R1's Loopback (Ensures the static route works).
